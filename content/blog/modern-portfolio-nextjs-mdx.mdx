# Building a Modern Portfolio with Next.js and MDX

Creating a portfolio website that stands out in today's competitive landscape requires more than just showcasing your projects. You need a platform that's fast, maintainable, and allows you to share your thoughts and experiences effectively.

In this comprehensive guide, we'll build a modern portfolio using Next.js 15 with the App Router, MDX for content management, and a suite of modern tools that will make your site both performant and delightful to use.

## Why This Tech Stack?

### Next.js 15 with App Router
- **Server-side rendering** for better SEO and performance
- **File-based routing** makes organizing pages intuitive
- **Built-in optimization** for images, fonts, and scripts
- **API routes** for dynamic functionality

### MDX for Content
- **Markdown with React components** - best of both worlds
- **Type-safe frontmatter** for metadata
- **Version control friendly** - your content lives in Git
- **No database required** - simpler deployment and maintenance

### Modern Developer Experience
- **TypeScript** for type safety and better DX
- **Tailwind CSS** for utility-first styling
- **Dark mode support** with next-themes
- **Global search** functionality with Fuse.js

## Content Management System

One of the key features of our portfolio is the content management system. We'll create a flexible system that can handle different types of content: blog posts, thoughts, projects, and talks.

### Content Types

Each content type has its own folder structure and frontmatter schema:

```typescript

  slug: string;
  title: string;
  description?: string;
  date: string;
  tags?: string[];
  published: boolean;
  content: string;
  readingTime?: string;
  // Blog-specific
  category?: string;
  // Project-specific
  techStack?: string[];
  liveUrl?: string;
  githubUrl?: string;
  // Talk-specific

```

### Content Loading

We'll create utility functions to load and parse our MDX content:

```typescript
import fs from 'fs';
import path from 'path';
import matter from 'gray-matter';

export function getContentByType(type: keyof ContentType): ContentItem[] {
  const typeDirectory = path.join(process.cwd(), 'content', type);
  
  if (!fs.existsSync(typeDirectory)) {
    return [];
  }

  const fileNames = fs.readdirSync(typeDirectory);
  const allContent = fileNames
    .filter(name => name.endsWith('.md') || name.endsWith('.mdx'))
    .map(name => {
      const fullPath = path.join(typeDirectory, name);
      const fileContents = fs.readFileSync(fullPath, 'utf8');
      const { data, content } = matter(fileContents);
      
      return {
        slug: name.replace(/\.mdx?$/, ''),
        content,
        ...data,
      } as ContentItem;
    })
    .filter(item => item.published)
    .sort((a, b) => new Date(b.date).getTime() - new Date(a.date).getTime());

  return allContent;
}
```

## Global Search Implementation

A standout feature of our portfolio is the global search functionality. Users can search across all content types using a keyboard shortcut (Cmd/Ctrl + K).

### Search Index

We use Fuse.js for fuzzy search capabilities:

```typescript
import Fuse from 'fuse.js';

const fuseOptions = {
  keys: [
    { name: 'title', weight: 0.3 },
    { name: 'description', weight: 0.2 },
    { name: 'content', weight: 0.1 },
    { name: 'tags', weight: 0.2 },
  ],
  threshold: 0.4,
  includeScore: true,
  includeMatches: true,
};

export function createSearchIndex(content: { [key: string]: ContentItem[] }) {
  const searchableContent: SearchResult[] = [];
  
  Object.entries(content).forEach(([type, items]) => {
    items.forEach(item => {
      searchableContent.push({
        ...item,
        type: type as SearchResult['type'],
      });
    });
  });
  
  return new Fuse(searchableContent, fuseOptions);
}
```

## Performance Optimizations

### Static Generation

All content pages are statically generated at build time for optimal performance:

```typescript
export async function generateStaticParams() {
  const posts = getContentByType('blog');
  return posts.map((post) => ({
    slug: post.slug,
  }));
}
```
